Request flow (high level):
1) HTTP request -> security middleware -> app/api/endpoints
2) endpoint dependencies (auth/step-up/throttle) in app/api/deps.py
3) service orchestration in app/services/*
4) persistence via app/crud/* and app/db/models/*
5) response shaping via app/schemas/* and app/utils/*
6) centralized error envelope from app/core/errors.py

Current security posture:
- JWT access auth with token_version revocation on logout/password reset
- Refresh token rotation stored hashed in DB
- Step-up PIN for destructive admin actions (shared env PIN; per-user MFA is future work)
- Auth/payment failure rate limits via Redis when REDIS_HOST is set
- Public endpoint request throttles (contact, checkout, PLP, tracking)
- OTP dev echo only when DEBUG=True and OTP_DEV_ECHO=True
- Production config rejects weak PIN, wildcard CORS, missing Redis, OTP echo
- Image URL validation blocks private/internal hosts (SSRF guard)
- Optional TRUSTED_HOSTS and ENFORCE_HTTPS middleware
- MAX_REQUEST_BODY_BYTES rejects oversized payloads early
